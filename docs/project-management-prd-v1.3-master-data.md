# 项目管理模块 PRD V1.3 - 主数据互通版

## 1. 产品目标

项目管理模块作为工程管理系统的项目主数据源，统一承载项目基础信息、资料台账、合同金额、结算付款、变更签证，并向审计看板、资料证据库和待办异常队列提供一致的数据口径。

本版本目标是把“项目管理”和“审计看板”从两个独立页面升级为同一项目链路的两个工作视角：

- 项目管理维护项目主档案。
- 审计看板引用项目主档案，并维护审计阶段、审计金额、资料状态和审计记录。
- 项目资料中心统一归集项目资料与审计证据。
- 首页和审计页优先展示待处理事项与异常风险。

## 2. 角色与权限

| 角色 | 主要职责 | 权限边界 |
| --- | --- | --- |
| 管理员 | 配置字段、维护用户、查看全部项目和资料 | 可配置表格字段、发起审计、删除资料 |
| 编辑人员 | 维护项目、上传资料、推进审计流程 | 可新建/编辑项目、上传资料、发起审计 |
| 只读用户 | 查看项目、审计进度和资料 | 不可修改主数据和流程 |

## 3. 项目主数据

项目管理模块是项目主数据源，项目唯一标识为 `project_records.id`。审计项目通过 `audit_projects.project_id` 关联项目主档案。

核心字段包括：

- 项目编号、项目名称、建设单位、施工单位、项目负责人。
- 项目状态、结算状态、审计联动状态。
- 合同金额、送审金额、已付款金额、变更签证金额。
- 计划开始日期、计划完成日期、项目说明。
- 资料完整度、缺失资料数量。

业务规则：

- 同一项目只能关联一个有效审计项目。
- 从项目管理发起审计时，自动带入项目名称、编号、建设单位、施工单位、负责人、合同金额、送审金额、计划日期。
- 审计看板不作为项目基础信息的主要维护入口，只维护审计过程信息。

## 4. 审计阶段门

审计流程采用阶段门方式管理：

| 阶段 | 目标 | 默认关注字段 |
| --- | --- | --- |
| 报审待受理 | 确认项目可进入审计 | 送审单位、送审金额、资料状态、报审日期、缺失资料、受理人 |
| 一级初审 | 完成初步审核 | 初审负责人、初审金额、审减金额、争议事项 |
| 二级复审 | 完成复核确认 | 复审人、复审意见、复审金额、差异原因 |
| 定案结论 | 形成定案结果 | 定案金额、核减金额、定案日期、确认状态、结论附件 |
| 办结归档 | 完成资料归档 | 归档编号、归档时间、资料完整性、后续支付状态 |

每个阶段必须具备：

- 阶段负责人。
- 进入时间与计划完成时间。
- 必填资料状态。
- 阶段操作日志。
- 可配置的阶段表格字段。

## 5. 项目资料中心 / 审计证据库

资料证据库统一展示项目资料与审计附件。

资料必须关联项目，可选关联审计阶段、资料分类、业务事项。当前版本先支持：

- 按项目名称、资料类型、审计阶段、上传人、上传时间筛选。
- 文件预览、下载、删除、重命名。
- 资料完整性状态：待补充、已提交、已确认、需更正。
- 缺失资料同步进入待办异常队列。

后续版本可扩展 OCR、电子签章、资料审批和版本比对。

## 6. 待办与异常队列

首页和审计页展示统一待办，优先回答“我现在该处理什么”。

待办来源：

- 即将到期：计划完成日期 7 天内到期。
- 已逾期：计划完成日期已过且未归档。
- 资料缺失：必填资料不完整。
- 审批退回：项目、结算或审计状态为退回/驳回。
- 金额异常：送审金额或变更签证金额明显超出合同金额。
- 待确认结论：审计到定案阶段但尚未归档。

每条待办包含：

- 项目名称、来源模块、待办类型、责任人、截止时间。
- 处理建议、风险等级、状态。
- 快捷动作：查看项目、查看审计、补充资料。

## 7. 业务文案规范

用户端不得展示开发或运维术语。

| 禁用表达 | 推荐表达 |
| --- | --- |
| active | 进行中 |
| submitted | 报审待受理 |
| project_id | 项目关联编号 |
| record not found | 未找到相关项目记录 |
| 接口异常 | 数据加载失败，请稍后重试或联系管理员 |
| SQLite / API / 数据库 | 用户端不展示 |

## 8. 验收标准

- 项目管理中新建项目后，可从项目详情发起审计。
- 同一项目重复发起审计时，提示“该项目已进入审计流程”。
- 审计看板显示的项目名称、负责人、金额与项目主数据一致。
- 审计表格按阶段展示差异化字段。
- 资料证据库可同时查看项目资料与审计附件。
- 缺失资料、逾期、金额异常、待确认结论可进入待办队列。
- 管理员可配置审计表格字段，普通用户不可见配置入口。
- 构建、部署、生产健康检查通过。
