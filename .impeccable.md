# 造价结算审计拖拽看板 — 设计上下文

## Design Context

### Users

- **主要用户**：基建施工企业造价部门工作人员（造价员、造价主管、项目经理）。
- **使用情境**：在 1920px 办公大屏或常规桌面显示器上，长时间处理造价结算审计台账、跟踪项目审核进度、协调第三方事务所与业主内审。
- **核心任务**：
  1. 快速查看各阶段项目分布与待办事项；
  2. 通过拖拽看板推进项目流程，并识别非法流转；
  3. 通过筛选快速定位超期、争议、高优先级项目；
  4. 编辑项目金额、审计主体、时限等关键字段。
- **情感目标**：界面应传递**专业、可信、高效、沉稳**的情绪，让用户在高压审计周期中感到秩序与掌控感，而非焦虑。

### Brand Personality

- **三词人格**：专业 · 秩序 · 可信赖
- **视觉语调**：克制、清晰、有权威感；不花哨，但细节经得起推敲。
- **情感目标**：
  - 看到数据面板时，立刻感知到全局状态；
  - 拖拽卡片时，流程规则清晰可见；
  - 超期/争议项目能在 0.5 秒内被视觉捕捉。
- **反参考**：避免卡通化、过度圆润、荧光渐变色、无关装饰动画；不做深色发光赛博朋克风格。

### Aesthetic Direction

- **视觉方向**：现代化 B 端企业工具风格，参考腾讯云控制台、阿里云管理控制台、飞书多维表格、火山引擎控制台。
- **色彩**：以冷静的中性灰为基底，品牌主色采用偏政务/企业蓝的 **#2563EB**（primary-600），辅助色为深蓝 `#1E40AF`、浅蓝 `#EFF6FF`；语义色保持红（危险/超期）、橙（警告/一审）、绿（成功/办结）、青（信息/定案）。
- **排版**：使用系统字体栈（PingFang SC / Microsoft YaHei / system-ui），正文 14px，标题 16–24px，数字使用 tabular-nums 保证对齐。
- **空间**：以 4px 为基准，采用 8/12/16/20/24/32/40px 间距节奏；卡片内 12–16px 内边距； generous whitespace 与高密度数据展示平衡。
- **组件**：TDesign Web Vue3 组件库，保持原生组件风格但精细覆盖变量；卡片使用 8px 圆角、1px 浅灰边框、极淡投影；按钮以小尺寸为主，操作按钮分组清晰。
- **主题**：默认浅色办公模式，暂不提供深色模式（B 端办公场景以浅色为主）。
- **动效**：状态切换使用 150–200ms ease-out；hover 提升 1–2px 或边框色变化；避免回弹、旋转等娱乐化动效。

### Design Principles

1. **信息密度优先，但不压抑**：B 端看板需要一屏展示足够信息，但通过留白、分区、字体层级避免压迫感。
2. **状态即视觉**：优先级、阶段、超期、争议、退回必须使用明确且一致的颜色/图标/边框语义，一眼可读。
3. **操作触手可及**：新增、刷新、筛选、编辑、复制等高频操作放在肌肉记忆可达区域，减少鼠标移动距离。
4. **一致高于创新**：在 TDesign 组件规范内做微调，不引入与组件库冲突的异形样式，保证开发与维护成本可控。
5. **尊重用户专注**：减少非必要装饰、弹窗、通知；网络提示、空状态、错误状态使用清晰但不打扰的文案。
6. **可访问性底线**：正文与背景对比度 ≥ 4.5:1，交互元素尺寸 ≥ 32×32px，聚焦状态可见，图标配文字使用。
